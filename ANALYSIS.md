## 4. 分析の単位と前提（疑似反復に注意）

- **被験者内（within-subject）反復測定デザイン**。同一参加者が同じ相談文上で3手法を同条件で評価 → 手法比較は**対応あり**で行うと検出力が高い。
- 414 行をそのまま独立標本として扱うと**疑似反復（pseudoreplication）**になる：1参加者が9行を生むため、行は独立でない。
  - 簡便法：参加者ごとに各手法のスコアを平均し **n=46 の対応データ**にして検定（§0の数値はこれ）。
  - 厳密法：**線形混合モデル（LMM）**で `participant`（と `consultation`）をランダム切片に。順序尺度を厳密に扱うなら**累積リンク混合モデル（CLMM / 順序ロジット）**。
- 主要アウトカムは1本に決め打つ（多重性回避）。推奨：**5項目の合成スコア（平均）を主要評価項目**、5項目個別は副次。

---

## 5. アウトカムの定義

1. **合成スコア（主要）**：5項目の単純平均（1–5）。事前に**信頼性係数 Cronbach's α**を算出し、合成の妥当性を示す（α≥0.7目安）。
2. **項目別スコア（副次）**：relevance/usefulness/specificity/trust/intention の各5段階。
3. **最良選択（副次）**：`is_best`。相談文ブロックごとの3択 → 手法別の選択率。

---

## 6. 解析手順（この順で進める）

### Step 1. 前処理・品質チェック
- `_cleaned.csv` 読み込み → §3で手法復元 → §3の健全性チェック。
- 低品質回答の検出（注意チェック設問は無いので代替指標で）：
  - **ストレートライン**：1行で5項目全て同一＝79/414行。9行すべて同一の参加者＝0名（＝全消しは不要）。
  - 参加者単位で「9行とも全項目同一値」「分散ゼロに近い」者をフラグし、感度分析（除外有無で結論が変わるか）。
- 除外したら理由と件数を必ず記録（再現性）。

### Step 2. 記述統計
- 手法 × 項目の平均・SD・SE・中央値、合成スコアの分布。
- **積み上げリッカート図**（手法別に1–5の割合）で分布の形（polarizing か）を見る。
- `is_best` の手法別カウント／割合。

### Step 3. 主要検定（合成スコア）
1. **オムニバス**：3手法の差を Friedman 検定（対応あり3群・ノンパラ）。LMM 派なら手法主効果の尤度比検定。
2. **事後ペアワイズ**：proposed vs 各ベースライン、reranking vs vanilla を Wilcoxon 符号順位（or 対応t）。
3. **多重比較補正**：Holm（または Bonferroni）。
4. **効果量**：対応 t なら Cohen's `dz`、ノンパラなら rank-biserial / Cliff's δ。点推定だけでなく**95%信頼区間**を併記。

### Step 4. 最良選択（is_best）
- 手法別選択率を一様（1/3）と比較：χ²適合度、または対応構造を活かすなら Cochran's Q。
- 実測 proposed 74 / vanilla 33 / reranking 31（138択）。proposedが54%＝偶然(1/3)から大きく外れる。χ²で有意になる見込み。

### Step 5. 項目別（確認）
- 5項目すべてで proposed が最上位（具体性 specificity が最大の差：4.29 vs 3.80/3.81）。**「具体性で勝つ」が提案手法の強みの核**と読める。多重比較は Holm/FDR で。

### Step 6. サブグループ／調整変数（頑健性の確認＝proposed優位はどこでも崩れないか）
proposed が全体で勝っているので、目的は「優位の在処探し」ではなく**頑健性チェック＋強み/弱みの記述**：
- **相談文別**（A/B/C）：実測 全相談で proposed 最上位（A 4.10 / B 4.01 / C 4.16）。崩れない。
- **家族人数別**（f1–f4）：実測 f1=4.20・f2=3.97・f4=4.27 で proposed 明確に首位、**f3 のみ拮抗（proposed 3.90 vs reranking 3.89, vanilla 3.57）**。f3で詰まる理由は要考察（支出設定が3人・4人グループの金額帯に切替わる境界。§後述の支出設定差）。
- **支出ラベル別**（ins/com/fod の L/H）：DESIGN.md §1 の既知のproposedバグ（低支出を「高すぎる」と誤読）が L 群でスコアを下げていないか。L/H 別に proposed の勝ち幅を見て、バグの影響を定量化。
- **事前属性別**：`fp_experience`（ある n=7 は小さい：proposedの優位が薄まる傾向＝FP経験者は辛口？）、`fp_trust`、`manages_finance`。n が小さい層は記述に留め確証に使わない。
- 厳密にやるなら LMM に `method × moderator` の交互作用項を入れて検定（交互作用が非有意＝「どの層でも提案手法が効く」という頑健性の主張に使える）。

> **支出設定が家族人数で変わる点（解析・考察で重要）**：L/H は相対水準ラベルで、絶対額は家族グループで異なる（`expense_patterns.md`）。
> 1・2人グループ：保険 1万/3万・通信 1万/2万・食費 3万/5万。3・4人グループ：保険 3万/5万・通信 2万/3万・食費 5万/7万。
> つまり **f1/f2 の insH（3万）と f3/f4 の insL（3万）は同額**。proposedのバグ（≈3万を一律「高い」と誤読）が family×L/H で非対称に効く可能性があり、f3 の拮抗の一因かもしれない。サブグループ解釈時に必ず考慮。

## 9. 「優位性を示す」ための論文化戦略

proposed が全項目・全相談文・最良選択・ほぼ全サブグループで有意に優位（§0）。主張は明快なので、論点は**「いかに頑健・誠実に示すか」**：

1. **主張のコア**：「提案手法はベースライン2手法より、5評価軸すべてで有意に高評価（dz≈0.5＝中効果）」。特に**具体性**で差が最大＝提案手法は「具体的なアドバイス生成」に強い、というストーリー。
2. **頑健性で固める**：LMM（参加者・相談文をランダム効果）で手法主効果が有意、かつ `method×相談文`・`method×family` の交互作用が非有意（＝条件に依らず優位）を示す。f3 の拮抗だけは正直に記述。
3. **ベースライン間に差がない点も誠実に書く**：reranking ≈ vanilla（検索の高度化単体では効かず、提案手法の枠組みが効いている、という対比に使える）。
4. **限界（査読対策に先回り）**：
   - proposed の既知バグ（低支出を「高すぎる」と誤読、DESIGN.md §1）→ それでも勝っている＝下振れ要因がある中での優位、と前向きに。L/H 別の感度分析で影響量を提示。
   - 属性・相談文は自己選択（網羅されていない）＝一般化の限界。
   - n=46・FP経験者 n=7 など小サンプル層は確証に使わない。
5. **任意の上積み**：効果量と現サンプルから検出力は十分そうだが、セル網羅をPhase2（DESIGN.md §11）で補えば一般化が強まる。

## 10. 再現手順（実行はユーザー）

分析スクリプトを起こす場合は **codex 経由で作成**し、実行はユーザーが行う（プロジェクト方針）。最低限の流れ：

1. `_cleaned.csv` を読み、§3の**固定対応（0=vanilla/1=reranking/2=proposed）**で手法列を付与。**`method_keymap.json` は使わない。**
2. 検証として `web/data/advice.json` の本文を `outputs/advice/<method>/*.txt` と突合し、固定対応が正しいことを毎回アサート（nomatch=0）。§3健全性チェック → 失敗なら停止。
3. 参加者単位集約（n=46）＋ロング（414）の両方を用意。
4. §5 アウトカム算出（合成・項目別・is_best、Cronbach's α）。
5. §6 検定（Friedman→Wilcoxon＋Holm、効果量＋95%CI、必要に応じ LMM/CLMM）。
6. §7 図表出力。除外・補正・探索の別を明記したレポートに。

> 検定・混合モデルは `pandas` + `scipy.stats`、LMM/CLMM は `statsmodels`（or R の `lme4`/`ordinal`）が手堅い。
